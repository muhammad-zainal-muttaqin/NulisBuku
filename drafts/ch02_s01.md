## Fit dan Transform: *Pipeline* sebagai Kontrak Pelatihan dan Inferensi

Dalam *machine learning*, terdapat aturan baku yang pantang dilanggar: transformasi data menuntut konsistensi absolut antara tahap pelatihan dan tahap inferensi. Model mengharapkan bahwa fitur dari data masa depan akan diubah menggunakan aturan dan metrik skala yang persis sama dengan data pelatihannya. Konsistensi ini adalah inti dari *praktik pipeline yang benar*.

Kunci utama untuk menjaga konsistensi ini terletak pada mekanisme *fit* dan *transform*. Tahap *fit* adalah proses pengamatan dan perhitungan parameter dari sebuah *dataset* pelatihan. Pada tahap ini, sistem melihat data latih untuk menghitung rata-rata, standar deviasi, atau memetakan rentang kategori. Selanjutnya, fungsi *transform* beroperasi sebagai tahap pengaplikasian parameter *fit* pada data observasi untuk menghasilkan representasi akhir.

Aturan utamanya sangat ketat: perhitungan parameter pada tahap *fit* hanya boleh dilakukan menggunakan data pelatihan. Setelah parameter tersebut dikunci, operasi *transform* menggunakan acuan yang sama persis untuk mengubah data pelatihan, validasi, maupun kumpulan data inferensi kelak.

Sebagai ilustrasi, misalkan kita melakukan standardisasi pada sekumpulan data pelatihan. Tahap *fit* menemukan bahwa fitur tersebut memiliki rata-rata 5,0 dengan standar deviasi 2,0. Beberapa bulan kemudian, sekumpulan data inferensi baru tiba. Bisa saja rata-rata aktual dari data baru tersebut adalah 7,0. Di sinilah letak ujiannya: kita wajib mengurangi setiap nilai pada data inferensi dengan 5,0 dan membaginya dengan 2,0. Acuan dari data pelatihan ini mutlak dipertahankan.

Jika kita malah menghitung ulang rata-rata pada populasi data inferensi (misalnya akibat keliru memanggil metode `fit_transform` pada data baru tersebut), makna fitur akan bergeser dari standar aslinya. Model akan salah mengartikan besaran nilainya. Fenomena ini dikenal dengan istilah *training-serving skew*, yaitu sebuah kesalahan fatal yang membuat prediksi model menjadi tidak valid di lingkungan produksi meskipun metrik akurasi pelatihannya terlihat sangat tinggi.

Mengingat manipulasi transformasi manual rentan terhadap kelalaian, kita perlu mengamankan proses ini ke dalam sebuah *pipeline*. *Pipeline* adalah rangkaian operasi pemrosesan data dan algoritma yang disatukan menjadi satu objek.

Sebuah *pipeline* secara formal membungkus kontrak *fit* dan *transform* ini. Saat dilatih, *pipeline* memastikan pencarian parameter terisolasi murni pada himpunan data latih. Setelahnya, *pipeline* secara otomatis menyalurkan parameter bawaan tersebut ke tahap *transform* untuk menangani setiap data baru. Lewat mekanisme ini, integritas keseluruhan tahap rekayasa fitur akan senantiasa terjaga.
