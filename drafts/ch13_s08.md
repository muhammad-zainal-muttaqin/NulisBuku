## Data Leakage pada Pemisahan Graf

Pemisahan data menjadi partisi pelatihan dan partisi uji pada data tabular adalah proses yang lugas, karena setiap baris diasumsikan saling bebas. Di dalam data graf, asumsi kebebasan ini tidak berlaku. Entitas saling terhubung melalui sisi, dan keterhubungan inilah yang menjadi inti masalah. Jika kita membagi simpul secara acak ke dalam set pelatihan dan set uji, banyak sisi akan melintasi batas pemisahan tersebut. Kondisi ini menciptakan celah besar bagi terjadinya kebocoran informasi atau *target leakage*.

Kebocoran pada graf terjadi secara halus melalui mekanisme agregasi ketetanggaan. Ketika kita merancang fitur berbasis struktur, atau ketika sebuah *GNN* melakukan proses *message passing*, informasi mengalir dari satu simpul ke simpul lainnya melalui sisi yang ada. Misalkan simpul A berada di set pelatihan dan terhubung langsung dengan simpul B di set uji. Fitur dari simpul B akan ikut diagregasi untuk memperbarui representasi simpul A selama pelatihan berlangsung. Akibatnya, model secara tidak langsung telah melihat data uji. Saat tiba waktunya mengevaluasi simpul B, model memberikan prediksi yang sangat akurat bukan karena representasinya kuat, tetapi karena informasi dari simpul tersebut telah mencemari proses pelatihan.

Desain pemisahan pada graf membedakan dua skenario evaluasi: pembelajaran *transductive* dan pembelajaran *inductive*. Pada skenario *transductive*, struktur graf secara keseluruhan terlihat selama fase pelatihan, namun label untuk simpul uji disembunyikan. Model membentuk representasi dengan melihat topologi jaringan secara utuh. Pendekatan ini rentan terhadap kebocoran fitur jika agregasi ketetanggaan tidak dibatasi dengan hati-hati. Sebaliknya, skenario *inductive* mensyaratkan model untuk dievaluasi pada graf atau subgraf yang benar-benar terisolasi. Set uji dipisahkan secara fisik dari set pelatihan tanpa ada sisi yang menghubungkan keduanya, memaksa model untuk menggeneralisasi pola ke struktur yang benar-benar baru.

Risiko kebocoran ini sangat jelas terlihat pada kasus deteksi penipuan keuangan. Bayangkan kita membangun *GNN* untuk mengidentifikasi akun palsu dalam sebuah jejaring transaksi. Jika pemisahan dilakukan secara acak, sebuah akun penipu di set pelatihan mungkin memiliki riwayat transfer langsung ke akun di set uji. Proses *message passing* akan menarik sinyal penipuan tersebut melintasi sisi yang menghubungkan keduanya. Akurasi model akan terlihat luar biasa tinggi pada saat pengujian. Akurasi ini bersifat artifisial. Di dunia nyata, model akan gagal karena koneksi langsung semacam itu belum tentu tersedia pada saat inferensi.

Menerapkan *praktik pipeline yang benar* pada data graf menuntut isolasi yang kaku sejak tahap awal pembuatan fitur. Untuk evaluasi *inductive*, memutus sisi penghubung antara subgraf pelatihan dan subgraf uji sebelum proses agregasi berjalan adalah langkah wajib. Pendekatan alternatif yang sering digunakan adalah pemisahan temporal. Pada jejaring yang tumbuh seiring waktu, model dilatih menggunakan struktur graf hingga batas waktu tertentu, dan dievaluasi hanya pada simpul atau sisi yang muncul setelah batas waktu tersebut. Isolasi ini memastikan fitur graf direkayasa hanya dari informasi yang benar-benar tersedia secara historis.
